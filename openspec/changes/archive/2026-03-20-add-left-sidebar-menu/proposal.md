## Why

当前的浏览器导航应用缺乏一个持久化的导航入口，用户只能通过页面上的链接进行跳转，操作效率较低。同时，随着功能的扩展，需要一个统一的侧边栏来组织导航、快捷功能、工具箱等多种入口，提升用户体验和功能可扩展性。

## What Changes

- 新增左侧可折叠菜单组件，支持多级菜单结构
- 实现菜单的展开/收起动画效果
- 支持菜单项的自定义配置（导航链接、功能入口、分隔符等）
- 添加菜单持久化状态（记住用户的展开/收起偏好）
- 为后续功能扩展预留插槽（搜索、历史记录、工具箱等）

## Capabilities

### New Capabilities
- `sidebar-menu`: 提供可配置的左侧菜单系统，支持多级结构、图标、徽章等丰富展示形式，具备响应式设计和状态持久化能力
- `sidebar-layout`: 提供左侧菜单与主内容区域的布局容器，处理响应式适配和移动端交互

### Modified Capabilities
- 无需修改现有能力，这是新增功能

## Impact

- **UI/UX**: 引入新的左侧菜单交互模式，需要考虑与现有页面布局的协调
- **依赖**: 可能需要引入图标库（如 Lucide React 或 Heroicons）
- **数据结构**: 需要定义菜单配置的数据结构（JSON 或 TypeScript 接口）
- **移动端**: 需要考虑小屏幕上的菜单交互（抽屉式或底部导航）
