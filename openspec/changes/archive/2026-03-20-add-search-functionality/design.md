## Context

当前浏览器导航项目是一个纯前端应用，使用 HTML、CSS 和 JavaScript 实现。网站数据存储在 localStorage 中，页面按分组显示网站列表。

现有技术栈：
- HTML5：页面结构
- CSS3：样式和布局
- Vanilla JavaScript：应用逻辑和 DOM 操作
- localStorage：数据持久化

当前限制：
- 随着网站数量增加，用户难以快速找到目标网站
- 缺乏搜索和过滤功能
- 需要保持轻量级，不引入额外依赖

## Goals / Non-Goals

**Goals:**
- 提供实时搜索功能，支持按网站名称和 URL 过滤
- 实现搜索结果高亮显示
- 支持键盘快捷键（"/"键）快速聚焦搜索框
- 保持响应式设计，适配不同屏幕尺寸
- 维持现有功能的完整性和性能
- 代码简洁，不引入额外依赖

**Non-Goals:**
- 不实现模糊搜索或高级搜索语法
- 不实现搜索历史记录
- 不改变现有的数据存储结构
- 不在管理页面添加搜索功能（仅限导航页）

## Decisions

### 1. 实时搜索实现方式
**决策：** 使用 input 事件监听器实现实时搜索，配合 Array.filter() 进行数据过滤

**理由：**
- input 事件在每次输入时触发，提供即时反馈
- Array.filter() 是原生 JavaScript 方法，性能良好
- 不需要防抖（debounce），因为数据量通常不大（一般少于 100 个网站）
- 相比 keyup 事件，input 事件能捕获所有输入方式（包括粘贴、自动完成等）

**备选方案：**
- 使用 keyup 事件：缺点是无法捕获粘贴等操作
- 使用防抖：优点是减少不必要的渲染，但增加复杂度；当前数据规模下不需要

### 2. 搜索高亮实现
**决策：** 使用正则表达式配合 innerHTML 实现高亮

**理由：**
- 使用 String.replace() 和正则表达式可以简单高效地替换匹配文本
- 使用 `<mark>` 标签实现高亮，语义化且易于样式化
- 不需要额外的 DOM 操作库

**备选方案：**
- 使用第三方库：增加依赖，不符合轻量级原则
- 创建多个 span 元素：增加 DOM 复杂度

### 3. 搜索框位置
**决策：** 将搜索框放置在页面顶部标题下方，工具栏上方

**理由：**
- 位置显眼，用户容易发现
- 不干扰现有的导航结构
- 在移动端也能保持良好的可见性

**备选方案：**
- 放置在侧边栏：当前页面没有侧边栏结构
- 使用浮动搜索框：可能遮挡内容，实现复杂度高

### 4. 键盘快捷键实现
**决策：** 使用 document 级别的 keydown 事件监听器

**理由：**
- 可以在任何位置触发搜索
- 需要检查焦点状态，避免在输入框中时触发
- 实现简单，不干扰现有键盘操作

**备选方案：**
- 使用 accesskey 属性：浏览器兼容性不一致，快捷键显示不直观

### 5. 响应式设计
**决策：** 使用 CSS Flexbox 和媒体查询实现响应式搜索框

**理由：**
- 与现有页面布局系统一致
- 简单可靠，兼容性好
- 可以在不同屏幕尺寸下调整搜索框宽度和样式

### 6. 状态管理
**决策：** 不引入状态管理库，使用简单的 JavaScript 变量管理搜索状态

**理由：**
- 搜索状态简单（当前搜索词、过滤后的网站列表）
- 使用现有变量管理模式，保持一致性
- 避免过度设计

## Risks / Trade-offs

### 性能风险
**风险：** 当网站数量很大（数百个）时，实时搜索可能导致性能问题
**缓解措施：**
- 当前使用 localStorage，数据量通常不会很大
- 如果未来需要支持大量网站，可以考虑添加防抖或虚拟滚动

### XSS 风险
**风险：** 使用 innerHTML 插入高亮标记时，如果搜索词包含 HTML 标签可能导致 XSS
**缓解措施：**
- 对用户输入进行 HTML 转义
- 使用文本节点操作而非直接的 HTML 插入
- 只对已知安全的内容使用 innerHTML

### 可访问性
**风险：** 搜索高亮可能影响屏幕阅读器体验
**缓解措施：**
- 确保高亮标记不影响语义结构
- 测试屏幕阅读器兼容性
- 提供足够的颜色对比度

### 兼容性
**风险：** `<mark>` 标签在旧浏览器中可能不支持
**缓解措施：**
- 提供降级样式（使用背景色）
- 目标浏览器通常都支持 `<mark>` 标签

## Migration Plan

### 部署步骤
1. 修改 `index.html`：添加搜索框 HTML 结构
2. 修改 `app.js`：添加搜索逻辑和事件监听器
3. 修改 `style.css`：添加搜索框和高亮样式
4. 测试：在不同浏览器和设备上测试功能

### 回滚策略
- 由于是纯前端增强功能，可以直接删除相关代码回滚
- 不涉及数据结构变更，无数据迁移需求
- 可以通过功能开关（注释代码）快速禁用

## Open Questions

1. **搜索高亮样式**：使用黄色背景还是其他颜色？
   - 建议：使用与主题一致的柔和颜色，确保文字可读性

2. **无搜索结果时的提示**：是否提供"清除搜索"按钮？
   - 建议：先实现基本提示，根据用户反馈决定是否添加按钮

3. **搜索框宽度**：在桌面端是否需要最大宽度限制？
   - 建议：设置合理的最大宽度，避免搜索框过宽影响美观
