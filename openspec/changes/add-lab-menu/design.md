## Context

当前应用有首页和管理两个一级菜单，使用平铺的菜单结构。系统支持两层菜单（一级 + 二级子菜单），但目前未使用子菜单功能。

用户需要一个实验区域来学习 React 和编写 demo，需要一个独立的空间不影响现有功能。

## Goals / Non-Goals

**Goals:**
- 添加"实验室"一级菜单，支持子菜单展开
- 利用现有的两层菜单结构，在实验室下创建实验页面
- 提供灵活的结构，方便后续添加新的实验页面

**Non-Goals:**
- 不改变现有的菜单组件架构
- 不引入新的依赖库
- 不修改现有的首页和管理页面

## Decisions

### 菜单结构设计

使用现有的 `MenuItem.children` 属性实现两层菜单：

```typescript
{
  id: 'lab',
  label: '实验室',
  icon: Flask,
  children: [
    { id: 'lab-demo1', label: 'Demo 1', path: '/lab/demo1' },
    // 后续可继续添加
  ]
}
```

**理由**：
- 利用现有架构，无需修改组件
- 符合用户需求（二级菜单放实验页面）
- 与现有 `SubMenu` 组件完美配合

### 图标选择

使用 `lucide-react` 的 `Flask` 图标表示实验室。

**理由**：
- Lucide 是项目已有的图标库
- Flask 烧瓶图标语义清晰，表示实验/实验室
- 与现有 Home、Settings 图标风格一致

### 路由设计

实验页面路由使用 `/lab/*` 前缀：

```
/lab/demo1
/lab/demo2
/lab/react-hooks-test
...
```

**理由**：
- 路由前缀清晰，便于识别实验页面
- 方便未来添加权限控制或功能开关
- 符合 RESTful 风格

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 实验页面质量参差不齐 | 实验室内容仅供学习，不影响生产页面 |
| 菜单过长导致拥挤 | 默认折叠实验室菜单，按需展开 |
| 实验页面路由可能冲突 | 使用 `/lab/` 前缀隔离 |
